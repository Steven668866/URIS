# 🎥 摄像头功能快速开始

## 📹 如何使用摄像头录制功能

### 第一步: 安装依赖

确保已安装 OpenCV:

```bash
pip install opencv-python numpy
```

或使用 requirements.txt:

```bash
pip install -r requirements.txt
```

### 第二步: 启动应用

```bash
streamlit run app.py
```

### 第三步: 使用摄像头

1. **测试摄像头**
   - 在左侧边栏找到 "📹 摄像头实时录制"
   - 点击 "📹 测试摄像头" 按钮
   - 确认看到摄像头画面，说明摄像头正常工作

2. **录制视频**
   - 选择录制时长（推荐 10-15 秒）
   - 点击 "🎬 开始录制" 按钮
   - 在摄像头前表演或展示你想要分析的内容
   - 等待录制完成（会显示进度条）

3. **提问交互**
   - 录制完成后，视频会自动加载
   - 在聊天框中输入问题，例如：
     - "这个视频中我在做什么？"
     - "请详细描述视频中的动作"
     - "视频中的背景是什么样的？"
   - 模型会分析视频并回答你的问题

### 第四步: 多次录制

- 可以随时点击 "🎬 开始录制" 录制新视频
- 新视频会替换旧视频
- 可以选择是否清除之前的对话历史

## 💡 使用技巧

### 录制建议

1. **光线充足**: 确保环境光线充足，画面清晰
2. **稳定拍摄**: 尽量保持摄像头稳定，避免晃动
3. **合适距离**: 保持适当距离，确保主体完整入镜
4. **时长适中**: 10-15秒是最佳时长，既能展示内容又不会太长

### 提问技巧

1. **具体明确**: 问题越具体，回答越准确
2. **分步提问**: 可以多轮对话，逐步深入
3. **描述需求**: 如果需要详细分析，可以说明"请详细描述"

### 性能优化

1. **选择合适时长**: 
   - 简单场景: 5-10秒
   - 复杂场景: 10-20秒
   - 避免超过 30秒（处理较慢）

2. **清理历史**: 
   - 长时间使用后点击"清除对话历史"释放内存

3. **调整参数**:
   - 如不需要长回答，可降低 "Max New Tokens" 到 512-1024

## ⚙️ 摄像头设置

### Mac 用户
- 首次使用需要在 "系统偏好设置 > 安全性与隐私 > 摄像头" 中授权
- 允许 Terminal 或 Python 访问摄像头

### Windows 用户
- 确保摄像头驱动已安装
- 检查设备管理器中摄像头状态

### Linux 用户
- 确保有摄像头访问权限
- 可能需要添加用户到 video 组: `sudo usermod -a -G video $USER`

## 🐛 故障排除

### 问题: "无法打开摄像头"

**解决方法**:
1. 检查摄像头是否被其他应用占用（如 Zoom、Skype）
2. 关闭其他使用摄像头的应用
3. 检查系统隐私设置
4. 尝试重启应用

### 问题: "录制的视频很暗"

**解决方法**:
1. 增加环境光线
2. 调整摄像头角度
3. 检查摄像头设置（如有调节选项）

### 问题: "录制卡顿或掉帧"

**解决方法**:
1. 关闭其他占用CPU的程序
2. 选择较短的录制时长
3. 降低系统负载

## 📊 性能参考

| 录制时长 | 文件大小 | 处理时间 | 推荐场景 |
|----------|----------|----------|----------|
| 5秒 | ~2MB | ~3秒 | 快速测试、简单动作 |
| 10秒 | ~4MB | ~5秒 | 标准使用、单一动作 |
| 15秒 | ~6MB | ~7秒 | 复杂动作、多步骤 |
| 20秒 | ~8MB | ~10秒 | 详细场景、长动作 |
| 30秒 | ~12MB | ~15秒 | 完整活动、全流程 |

*注: 实际时间取决于硬件配置*

## 🚀 高级用法

### 连续录制多个片段

1. 录制第一个片段，提问并获得回答
2. 点击 "🎬 开始录制" 录制第二个片段
3. 可以选择清除历史或保留历史继续对话

### 对比不同视频

```
第一次录制: 展示动作A
提问: "这个动作是什么？"

第二次录制: 展示动作B  
提问: "这个动作和之前的有什么不同？"
（选择不清除历史，模型会记住上下文）
```
